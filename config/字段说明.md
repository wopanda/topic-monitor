# 配置字段说明

## 主文件
- 默认文件名：`config/topic-monitor-config.json`
- 首次使用：从 `config/topic-monitor-config.example.json` 复制一份出来再改

## 搜索策略

对读者来说，默认只需要记住一件事：
- **当前版本默认用 Bocha**

运行前请先设置：

```bash
export BOCHA_API_KEY='你的 Bocha API Key'
```

重点配置项：
- `search.provider`：固定为 `bocha`
- `search.endpoint`：Bocha API 地址
- `search.freshness`：时间范围
- `search.summary`：是否要摘要

## 读者最需要先改的字段

### topics[0].name
日报里展示的主题名。

### topics[0].keywords
真正驱动搜索的关键词数组。

### topics[0].preferredDomains
希望优先看的站点域名，例如 `github.com`、`larksuite.com`。

### topics[0].blockedDomains
明确要压掉的域名。

### output.finalItems
精选条数。

### output.watchItems
观察池条数。

### filters.excludeKeywords
排除词，例如广告、推广。

## 定时发布相关字段（推荐方案）

### schedule.time
默认定时发布时间，例如 `09:00`。

### schedule.timezone
默认时区，例如 `Asia/Shanghai`。

### schedule.enabled
是否默认建议开启定时发布。建议保持 `false`：
- 先跑通并看样稿
- 再询问用户是否开启定时发布

### delivery.channel
默认投递渠道。当前建议默认 `feishu`。

### delivery.target
默认投递目标。可留空；真正创建 cron 时再传，例如：
- `user:ou_xxx`
- `chat:oc_xxx`

### delivery.method
定时发布实现方式。当前建议固定为 `openclaw-cron`。

### delivery.askAfterManualRun
手动跑完一次日报后，是否默认追问用户“要不要设置定时发布”。建议保持 `true`。

## 实操建议

- 先手动运行一次：`bash scripts/topic-monitor-run.sh`
- 样稿确认后，再询问用户是否开启定时发布
- 用户确认后，使用：`scripts/create-openclaw-cron.sh`

## 已做实测

### A. 直接调用 Bocha 原始接口
已实测：
```bash
curl --location 'https://api.bochaai.com/v1/web-search' \
  --header 'Authorization: Bearer $BOCHA_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "query": "OpenClaw",
    "freshness": "oneYear",
    "summary": true,
    "count": 5
  }'
```
结果：
- 接口可通
- 返回结构主数据在 `data.webPages.value`

### B. Bocha 适配器
已实测：
```bash
python3 scripts/search_bocha.py "OpenClaw 用例" --count 5 --pretty
```
结果：
- provider = `bocha`
- resolved_route = `bocha`
- used_provider = `bocha`
- 能拿到结果列表

### C. 完整运行入口
已实测：
```bash
export BOCHA_API_KEY='你的 Bocha API Key'
TOPIC_MONITOR_OUTPUT_DIR=./output bash scripts/topic-monitor-run.sh
```
结果：
- 运行入口已切到 Bocha
- 能生成日报文件

## 本轮关键修正

这次实际踩到的关键问题不是关键词，而是：
- **Bocha 的返回结果不在顶层 `webPages`**
- 而是在 **`data.webPages.value`**

前面第一次结果为空，就是因为这里取错了层级。
修正后，结果已能正常进入后续渲染链路。
